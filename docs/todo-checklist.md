# 기능 설명
    * 현장별 필요 서류 누락 방지 기능: 새 프로젝트가 열릴 때마다 어떤 서류가 필요한지 다시 판단하다가 누락하는 일이 없도록, 현장 개설시 미리 입력된 공종별 필수 서류 체크리스트에 맞게 서류를 챙겼는지 확인할 수 있는 기능
    * 템플릿 등록: 
        - 업로드 된 템플릿은 llm이 텍스트와 문서 틀을 그대로 html로 그려내어 관리되며, google docs에서 표를 그려 입력하는 것과 같이 입력된 내용에 따라 문서 틀의 높낮이나 너비가 달라질 수 있어야 한다.(높낮이를 달리 할지, 폭을 달리할지는 사용자 선택)
        - 빈 템플릿 문서를 업로드하면 해당 문서의 입력 항목을 미리 라벨링하여 다른 문서라도 같은 라벨로 관련 정보를 일괄 수정이 가능해야 함(라벨링 항목에는 서명도 포함되며 서명은 라벨과 함께 영역을 지정하면 해당 영역이 해당 라벨의 서명 영역이 되며, 서류가 달라도 같은 라벨은 같은 서명이 입력될 수 있게 되어야 합니다.)
            - 예를 들어 안전 책임(같은 담당자)가 일괄 서명할 일이 있다면 해당 담당자가 서명할 수 있는 문서 리스트가 등장하고 체크한 문서는 한 번에 정보를 확인하고 각 문서의 요약 내용 확인 후 서명할 수 있어야 함
    * 템플릿 추출: 값이 입력된 항목은 LLM이 문서를 읽고 템플릿을 만들어낼 수 있어야 함
    * 서류와 연계된 현장별 사진 라벨링 보관: 각 현장별 사진은 현장id와 라벨을 통해서 관리되며, llm이 사진을 보고 어떤 라벨이 붙어야 하는지 제안해서 필요 라벨을 누락하는 일이 없도록 해야 한다.
    * 일괄 정보 입력: 동일한 라벨을 가지고 있는 경우 선택한 문서는 내용을 일괄 입력|수정|삭제가 가능하다. (서명 포함)
    * 일괄 요청: 문서의 라벨을 선택하여 메일, 문자를 입력하면 해당 메일과 문제로 해당 부분만 제한적으로 입력할 수 있도록 문서 링크(템플릿구현을 html로 했듯, 문서 작성과 수정도 웹 페이지에서 가능해야 함)를 보낼 수 있어야 함
    * 변환 저장: 템플릿, 문서는 html로 구현되나 pdf, word문서(.docx), hwp 양식으로 수정 가능하게 저장할 수 있어야 한다.
