# Template Edit Speed Checklist

목표: `http://localhost:3001/templates/edit`의 상자 편집 캔버스에서 선택, 입력, 스타일 변경이 사용자 조작 후 100ms 이내에 화면에 반영되어야 한다.

측정 기준:
- 기준 페이지: `http://localhost:3001/templates/edit?templateId=0c6197e9-a221-46a9-9367-1e6bc84938c4`
- 측정 도구: Chrome DevTools의 실제 페이지 DOM 이벤트와 `performance.now()`
- `기존 소요시간`: 최적화 전 브라우저 실측값
- `해결 후 실제 소요시간`: 최적화 후 같은 방식으로 측정한 화면 반영 시간
- `timeout`: 스크립트 재현에서 기대 상태를 확인하지 못하고 제한 시간까지 대기한 항목

## 확인된 병목

- `applySelectionStylePatch`가 모든 스타일 변경 뒤 `syncDraftPreviewHtmlRef`, `syncSelectionStyleDraft`, `requestPreviewTextFit`을 즉시 실행했다.
- `syncDraftPreviewHtmlRef`는 전체 캔버스 HTML 추출과 히스토리 기록을 수행하므로 색상처럼 레이아웃과 무관한 변경에도 100ms를 넘겼다.
- `syncSelectionStyleDraft`는 선택 상자들의 `getComputedStyle`을 여러 번 읽기 때문에 복수 선택에서 비용이 더 커진다.
- 셀렉트 변경은 진행 상태/저장 상태/드래프트 상태를 한 이벤트 안에서 여러 번 갱신해 React 렌더를 즉시 유발했다.
- 비자동 텍스트 입력은 입력마다 전체 HTML 직렬화를 수행했다.

## 적용한 개선

- 스타일 DOM 적용과 저장용 HTML 동기화를 분리했다.
- 저장용 HTML 동기화, 패널 값 재계산, 텍스트 맞춤은 600ms 지연 큐로 합쳤다.
- 연속 조작 중에는 이전 지연 작업을 취소하고 마지막 변경만 동기화한다.
- 스타일 변경 이벤트에서 선택 저장 진행률/필드 저장 상태 갱신을 제거했다.
- 셀렉트 변경 이벤트에서는 스타일만 적용하고, 컨트롤 값 표시는 `requestAnimationFrame`으로 DOM 값을 보정한다.
- 비자동 텍스트 입력의 전체 HTML 직렬화를 즉시 실행하지 않고 지연 큐로 보낸다.

## 남은 재검증

- Chrome DevTools MCP가 `Transport closed`가 된 뒤 9223 포트가 응답하지 않아 최종 전체 매트릭스 재측정은 중단됐다.
- 다음 브라우저 연결 시 S-02, S-03, S-04, T-01, Z-01, A-01~A-03, 텍스트 스타일 시각 반영을 같은 체크리스트로 재측정해야 한다.
