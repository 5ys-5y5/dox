chrome-devtools 분리/복구 운영 문서 (반복 방지용)
작성일: 2026-05-13 KST

목적

- 다른 LLM이 기존 chrome-devtools를 사용 중이어도, 현재 Codex 세션에서 브라우저 디버깅을 안정적으로 복구한다.
- 동일 원인으로 같은 절차를 반복하지 않도록, 증상별로 단일 처리 루트를 고정한다.


핵심 원칙 (반드시 지킬 것)

1) 기존 LLM이 쓰는 Chrome/DevTools는 절대 종료하지 않는다.
2) 이 Codex 창은 설정 변경 후 반드시 재시작한다.
3) `mcp__chrome_devtools__`는 세션 시작 시 바인딩되므로, 세션 중간 설정 변경만으로는 붙지 않는다.
4) 실패하면 추측하지 말고 아래 “증상별 처리표”대로만 진행한다.


빠른 실행 파일

- 분리 Chrome(9223) + config 자동 갱신:
  - `docs/setup_split_chrome_devtools.command`
- 로컬 포트 접근 이슈 시 isolated 모드로 강제 전환:
  - `docs/setup_chrome_devtools_isolated.command`


증상별 처리표 (반복 방지)

A. `Transport closed` + config가 `--browser-url=http://127.0.0.1:9223`

- 원인:
  - 현재 Codex 실행 문맥에서 `127.0.0.1:9223` 접근이 불가한 상태(샌드박스/네트워크 정책 포함).
- 처리:
  1) `docs/setup_chrome_devtools_isolated.command` 실행
  2) 이 Codex 창만 재시작
  3) 재시작 후 `list_pages`로 연결 확인
- 주의:
  - 이 경우 `setup_split_chrome_devtools.command`를 반복 실행해도 같은 실패를 재현할 수 있다.


B. `browser profile in use`, `Use --isolated` 류 충돌 메시지

- 원인:
  - 동일 `user-data-dir`을 다른 세션이 점유.
- 처리:
  - `docs/setup_chrome_devtools_isolated.command` 실행 후 Codex 창 재시작.


C. 장기적으로 상태 유지(쿠키/세션) 필요

- 원인:
  - isolated는 임시 프로필이라 세션 상태 유지에 불리함.
- 처리:
  1) 별도 Chrome을 별도 프로필/포트로 직접 실행
  2) `~/.codex/config.toml`에서 `--browser-url`로 그 포트에 연결
  3) Codex 창 재시작
- 예시 포트/프로필:
  - `--remote-debugging-port=9223`
  - `--user-data-dir=$HOME/.cache/chrome-llm-b`


표준 복구 절차 (기본)

1) 먼저 이 문서의 “A”를 기준으로 판단한다.
2) 불확실하면 isolated 우선:
   - `docs/setup_chrome_devtools_isolated.command`
3) Codex 창 재시작.
4) 재시작 직후 `mcp__chrome_devtools__.list_pages` 성공 여부 확인.
5) 성공하면 브라우저 재현/검증 작업을 진행.


운영 체크리스트 (작업 시작 시 30초 점검)

- `~/.codex/config.toml`의 `mcp_servers.chrome-devtools.args` 확인
  - `--browser-url=...` 인지, `--isolated=true` 인지 확인
- 현재 세션이 재시작 이후인지 확인
- 첫 도구 호출로 `list_pages` 확인
- 실패 시 즉시 증상별 처리표 A/B 적용


이번 이슈에서 확정된 사실 (재발 방지 메모)

- `--browser-url=http://127.0.0.1:9223`로 고정된 상태에서, 현재 Codex 문맥은 로컬 포트 접근이 불가해 `Transport closed`가 발생할 수 있다.
- 이 상황에서는 split 스크립트 반복보다 isolated 전환이 우선이다.
- 설정 변경 후 재시작하지 않으면 MCP 바인딩이 유지되어 복구되지 않는다.


참고: 수동 설정 예시

`~/.codex/config.toml`에서 isolated 모드:

  [mcp_servers.chrome-devtools]
  command = "npx"
  args = ["-y", "chrome-devtools-mcp@latest", "--isolated=true"]
  startup_timeout_ms = 20000

`~/.codex/config.toml`에서 browser-url 모드:

  [mcp_servers.chrome-devtools]
  command = "npx"
  args = ["-y", "chrome-devtools-mcp@latest", "--browser-url=http://127.0.0.1:9223"]
  startup_timeout_ms = 20000
